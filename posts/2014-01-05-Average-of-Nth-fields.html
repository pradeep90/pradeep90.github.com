Average of Nth fields

Imagine you have the following task:

You want to find out the number of times you have used the function foo_bar in the Python files in the current directory.

How would you accomplish this?

Think about it for a second.

Ok, so you decide to use good ol’ grep

grep -c 'foo_bar' *.py

Here’s the output:

bar2.py:173
bar3.py:82
bar.py:43
foo2.py:1
foo_bar_baz2.py:74
foo_bar_baz3.py:32
foo_bar_baz.py:48
foo.py:8
foo.py3:33
test_bar2.py:30
test_bar3.py:22
test_bar.py:43

Cool. We’re almost done.

Now what?

I want to know the total. In fact, I would like the average.

How would YOU solve this? Really. In real life.

Adding up the numbers by hand is out of the question now.

Plus, it is not repeatable. If I change the files or want to count the occurrences in a different directory, we’re out of luck.

We want something that is:

  • easy
  • repeatable

Here’s one idea

awk -F: '{ sum += $2 } END { print sum / NR }'

You can figure out what it means even if you aren’t familiar with Awk.

Awk will iterate over each line and pattern-match based on the rules we gave.

Here’s the breakdown:

  • -F - the field separator
  • sum += $2 - the value of the 2nd field gets added to the variable sum (which is initially 0 by default)
  • END - At the end
  • NR - number of rows (~ lines) encountered
  • print sum / NR - print (sum / num of lines)

More on this stuff later

PS

Yes, I know you can do this by doing plain grep and counting the number of lines in the output :P
