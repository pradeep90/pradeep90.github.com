Practicing the Scientific Method

Dive in

Taboo the term Scientific Method. What do I really want to do?

I want to increase the number of times I predict outcomes accurately. Let’s just try it out quickly and then correct ourselves. Which experiments do I care about predicting?

Skills

Empiricism is one of the most important skills. I need to distinguish between the map and the territory and always seek empirical evidence. Next, I need to think in terms of anticipation-constraints instead of mere words. Then, I need to maintain hypotheses explicitly and update them on evidence. Lastly, I need to run hypothesis tests that distinguish well between my hypotheses.

Right now, I’m hitting a block because there seem to be so many things you can do (on the same sentence) and I don’t know where to start. Ok, Rapid Prototyping 101 - build a bare minimum hello world version. What can I not release without? I can deal with empiricism later. Anticipation-constraining is important, but right now I need something that will do work that cannot be replaced, a sine qua non if you will. What makes the scientific method the scientific method is the idea of differing predictions between hypotheses. Even the Bayesian updating on evidence can be done later. But testing things experimentally is the unique core of science, I think.

Ok. So, what can I test experimentally? What do I need to test experimentally? Well, what am I uncertain about? What can’t I explain and predict accurately right now?

Applying the Scientific Method

There is no such thing as the proper domain of science. Everywhere you are uncertain, you need to use science.

Ok, for our v1.0, we need to apply just the skill of testing hypotheses experimentally. Let’s do it places where we don’t usually apply the scientific method.

How would you apply it to essay-writing? Say you want to write an essay about a topic X. Well, actually, what you want is to find out the truth about topic X. You don’t just want to write down whatever untested thoughts you happen to have. So, you want to discover the truth about X. How can you do this scientifically? You need to come up with alternative hypotheses about X, look at their differing predictions, and run experiments.

Actually, I’m behaving like it’s hard to find places where I can apply the scientific method. I’ve been thinking fruitlessly for a few hours about where I can possibly use it. It’s not hard.

Remember, not every change is an improvement, but every improvement is a change. So, everywhere that I’m not satisfied with my performance, I can use the scientific method to find out what will work better and therefore what I should change. This has the positive effect of also being motivating - I care about improving these areas so I will take action more readily than in some dry abstract area about which I don’t give a damn.

Where am I dissatisfied?

I’m trying to set up some deployment pipelines so that I can get much better results in my projects. But it isn’t going well yet. I need to make changes there. Similarly, I want to practice thinking scientifically. Haven’t made much progress there. Also, I want to fix my sleep cycle - I’m sleeping late at night and waking up late. Not cool. Finally, I also want to improve my progress in the area of physical fitness - not fully happy with the results there.

Once you pose these as problems, it’s just obvious that I need to change things here. Of course I have to change things - the current way is not working! If I’m not getting results despite doing what I thought would work, it means my hypothesis is wrong.

Specific current example

What is my hypothesis about deployment pipelines and how can I falsify it? After my essay on cycle time, I started believing that if I had a dashboard showing some stats from my different projects, I would take more action and get those projects done faster. That has obviously not happened.

What is my alternative hypothesis? The simplest one seems to be that a dashboard won’t help at all, no matter what you put in it. To be more specific, if I display some stat on the dashboard, like my sleep hours, it won’t make me behave so that it improves within a few days.

How can I test my hypotheses experimentally? I need to look at their differing predictions. Where do they differ? Well, my hypothesis says that the dashboard will make a difference, and the other hypothesis says that it won’t, no matter what. So, let’s see if there is any stat at all that I can put on the dashboard and any number of times I can force myself to view the dashboard that will improve the stat within a few days.

Well, the continuous delivery guys say that you should view the feedback from your deployment pipeline so that you take action on it. What are my deployment pipelines? For now, essays on my website, my fitness, and sleep cycle. What kind of feedback from them would make me behave so as to improve the feedback?

For fitness, the number of hours of cardio and weight-lifting should help. They are things that I can change. For sleep cycle, the time at which I went to sleep, the time at which I woke up, and thus the amount of sleep I got. My bedtime is the one I can affect most, and also the question of whether I sit and watch videos in bed, which makes me stay up longer. For essays, I want to maximize the number of surprising and useful ideas I have come up with and explained clearly.

I need to do what matters. Show me whether I’m doing it or not.

Lessons

Look at alternative hypotheses. You will be surprised by how often you can’t even think of one.

Where are you not getting the results you want? You have the wrong hypotheses there.

How can you falsify your current hypothesis? Raise strong alternative hypotheses.

Other ideas

Essay-writing is one of the places where scientific thinking can help. I believe that if we wrote our essays in terms of hypotheses and empirical evidence that distinguished between them, then we would come to the correct answers more often, and more importantly not get waylaid by poor answers.

We routinely fail to notice what our beliefs forbid (if they do at all). That was the whole point of Eliezer’s sequence on Mysterious Answers to Mysterious Questions. If you predict that something will happen, you are necessarily predicting that the other thing will not. Also, we avoid testing our beliefs. We like to stay comfortable. Then, we also fail to collect empirical evidence - we just go by our naive thoughts. Even when we decide to think scientifically, we may not choose the most informative hypothesis tests.

Science vs Design?

One of my beliefs right now is that if we started thinking more scientifically, we would solve a lot more problems quickly. I need to drill down to the core of this and find out exactly when (or if) this is true, and how far scientific thinking can help me.

Let us look at the conventional hypothesis I differ from. How else do people suggest you should think?

In the sciences, they mostly already use the scientific method. So, we have no difference there. Where do people not use the scientific method? In programming, for example. In businesses too, I believe. In the arts - writing, singing, composing, acting, etc. When people are designing things, I think they tend not to use the scientific method directly (this is a testable prediction).

So, my hypothesis is that the scientific method is great for figuring out the rules in a domain - why something behaves the way it does. Once you roughly know the rules, I think you tend to use a different kind of process to design things.

In programming, you know the core rules of your language. And now you want to build a program that satisfies certain criteria. Hmmm… the problem is you don’t really know the final program you want. You cannot visualize all the requirements in your head, they are too numerous. Plus, they keep changing over time. So, maybe what you’re uncertain about here isn’t the working of the language, it’s what you want to build and how you want to build it.

The same holds for other areas of design, I think. In writing, you may know the basic rules of grammar, but you’re unsure about what you really want to say and how you want to say it. Similarly for composing music or painting, I suppose.

In physics, chemistry, and the other sciences, the problem was that we humans were unsure about the rules of the domain itself. We didn’t know how exactly gravity worked, or which chemicals would combine to do what, or what each part of our body did. The main aim there was to discover the rules of the domain - the laws of physics, the properties of chemicals, the anatomy of the human body.

If the scientific method (or Bayesian thinking) is theoretically the most efficient way of getting correct knowledge about the world, why do we not use it when designing things? Is it because of our cognitive limitations? Or is the nature of the problem itself different from those in physics or chemistry? In short, is there a difference between reduction and creation?

When designing, it is not the rules you’re unsure of, it is the product you want to design. You are unsure of the exact combination of building blocks that will pass the feedback test. You are even unsure of the exact feedback test you should use. In most projects, both the product and the feedback mechanism used evolve over time.

However, just because you’re unsure about a different thing doesn’t mean you should change the reasoning you use to get to the answer.

Just because I’m ignorant about the universe doesn’t mean that I’m unsure about how I should reason in the presence of my uncertainty. The laws for thinking with probabilities are no less iron than the laws that govern old-fashioned logic

– Harry Potter, HPMOR Chapter 17

I think we should use the scientific method no matter what we are uncertain about. Perhaps we can use some approximations if our mind can’t handle the full-blown thing.

What is the difference, though, between a usual science problem and a design problem? You have more control over the subject in the latter. In programming, for instance, you deal with something that is completely man-made, you have as much control as you want over the behaviour of the system. Perhaps this means there are far more variables to control for. There may be too much information to hold in your head - like when working on very large programs. Similarly, you may find it hard to make changes. It may also be really hard to come up with hypotheses in the first place.

However, that doesn’t mean you can just go ahead and do whatever you want. There is a right way to think under uncertainty, and every other way of thinking will be provably worse. If you don’t use the laws of probability to think, how else will you decide what to do?

That is the core of my argument here. There is a right way of thinking under uncertainty, and whenever we deviate or choose an ad hoc path, we will do worse. Of course, we may not be able to think perfectly, but even given our bounded resources, I claim that we can do better.

Insofar as our thought processes mimic the right way, you will get good results. We need not be fully conscious of what we’re doing. So, the aim isn’t to deliberately think in a scientific way at each step. The aim is to make our thought processes hew as closely to the ideal way of thinking as possible.

How do we make our thought processes match the ideal way of thinking as much as possible, in a variety of concrete domains?

Long arm of the law

You can’t protect your cherished beliefs by not letting them be tested. Hypotheses can be tested by anything they talk about, just like everything you say to a police officer can be used against you in a court. Their reach can be far subtler than we realize. So, you should aim to relinquish your hypotheses as soon as possible. Try to find as many places to falsify them as you can. Bring up strong alternative hypotheses and look at where they differ, so that you get to more and more predictive hypotheses.

Causal Model

Having a causal model is not optional. Without it you just have a bunch of predictions, so you will be severely penalized when you’re asked about a different random experiment. This is also what Eliezer was talking about in Mysterious Answers to Mysterious Questions. You need to have a causal model making the predictions. That is why everything will become super-clear and there will be no more confusion. For you will be able to see everything!

State-transition model is key here. Your hypothesis doesn’t just work with variables (dependent or independent or whatever). It works with the entire state. In the future, if I look at some other variable, your hypothesis should make correct predictions.

Science is so powerful because when you do get it right, you understand exactly how the states of the world unfold. There is no more mystery. You know how the system generates each of the variables you perceive. You also know how it will behave even after you change a bunch of parts inside it.

Science doesn’t work by waving wands and chanting spells, it works by knowing how the universe works on such a deep level that you know exactly what to do in order to make the universe do what you want.

– Harry Potter, HPMOR Chapter 7

Why do we care about having a causal model? Because the more you can predict how something behaves, the more you can tweak it to do what you want at low cost.

How does this affect the question about using the scientific method in other domains, like design? Is the causal model really complicated in those areas? Well, you can always approximate it. The point that you can’t use anything but the scientific method still stands.

Note that this is why you need to do a literature review before you go and do some new research. You need to understand the old causal model before you propose any extensions to it.

Now the question arises: in what language do you express the causal model? It is a program that you use to generate predictions for different experiments. Which language do you use?

Also, what you’re trying to do is approximate the actual working of the system, which is described completely in terms of interactions between atoms or quantum fields or whatever. That hypothesis is too huge for us to generate and too complex for us to handle, so we make some simplifying assumptions and build hypotheses on top of them. But the general principle stands, the closer you get to the actual algorithm that the universe is using to generate its states, the more predictive power you will have.

Taboo: No magic numbers

In programming, we have a rule: your code should not have any magic numbers embedded in it. The statement return foo * 25; will be dismissed immediately by any decent programmer. Why? Because there is no apparent reason for the existence of 25. Why is it 25, why not 26 or 24? It may represent the number of students in a class, but suppose next semester the number of students changes, how are you supposed to know what to change?

Similarly, your hypothesis should not have any magic numbers embedded in it. “Cockroaches are sensitive to electromagnetic radiation of frequency 50Hz” - that should raise a red flag immediately. Why are you talking about 50Hz? Why not 60Hz or 174Hz? What will happen in those cases? Sadly, your “hypothesis” doesn’t say anything there. A good hypothesis would tell us exactly how the behaviour of the cockroach changes with electromagnetic radiation, and so help us calculate its response for a range of frequencies.

Along the same line, your hypothesis should not have any unjustified magic effects. “If a plant receives fertilizer, then they will grow to be bigger than a plant that doesn’t receive fertilizer” - why are you talking about fertilizers in particular? Do you have any causal model that tells you how fertilizers will help plants grow bigger? Yes, you may have observed fertilizer helping plants several times, but that should just leave you puzzled because you have no clue why it is happening that way. For example, what will happen if I dump double the amount of fertilizer? Will the growth double? What if I only add a tiny bit, will the growth diminish?

What if I change the type of fertilizer - instead of phosphate, what if I use ammonia or whatever (I’m just pulling these out of my ass)? In fact, what makes a “fertilizer”? “Fertilizer” is just a label that people put on certain chemicals. What if you had a box labelled “Poison” and you give it to the plant and their growth doubled, would you say that Poison makes plants grow?

Let me resolve the confusion. We need to taboo key terms in our hypothesis so that we understand exactly why things are happening. Don’t use the term “electromagnetic radiation”: “Cockroaches are sensitive to qnthla” - see how silly that sounds? It doesn’t explain anything. In fact, go ahead and taboo “cockroaches” and “sensitive” too. Now, you’re forced to explain exactly how the radiation burns their antenna (or whatever; I’m making this up because I Am Not A Biologist) and how they try to escape antenna-burning. Now, you have several fronts on which this hypothesis is testable, all the better for it to be accepted strongly if it emerges unscathed. You might find some other species whose antenna is sensitive to radiation, or you can try some other ways of burning their antenna, whatever.

In short, don’t write your hypotheses in a natural language like English. Write them in the technical language of the domain, where you can’t escape out of tough situations by just hand-waving or using suggestive names.

(Thanks to Dr Paul Strode who taught me the difference between research hypotheses and mere predictions - web page; great presentation)

Fundamental Questions of Science

The core question is: do you have a causal model that generates predictions?

(You can think of this as a hypothesis: “I have a causal model about this domain”. The following experiments will test whether you really do.)

If you can’t handle changed conditions, you don’t really have a causal model. If you can’t explain why exactly you predicted this particular outcome, not another, you don’t really have a causal model.

What will happen if I change the conditions? Can your hypothesis handle the new case? Is your hypothesis a causal description or just a bunch of predictions for some specific cases? The fundamental question here is similar to that in programming - can you handle change? If you try to encode each of the special cases (what will happen if X happens, if Y happens, etc.), then your hypothesis will become very large and get penalized for complexity. You have to extract a common causal model that generates all of those special cases parsimoniously.

To help you see what conditions to change, use taboo. See through the labels that seem meaningful to you and talk in terms of the territory. Don’t say “food”, say “an apple” or “a rotting chicken leg”.

Another way to know if you really have a causal model: why do you predict exactly this particular outcome? Why not some other one? You should have a good explanation for why the world ended up in this state and not another. Your causal model should generate precisely this outcome. Else, all you have is just a prediction, and when your hypothesis is questioned about a different experiment, it will have nothing useful to say and thus get downshifted. “Fertilizer makes plants grow bigger” - why don’t they make plants grow smaller? Why did you pick an arbitrary position?

Next, why are you running this particular experiment? What is the alternative hypothesis? Is this the most informative differing prediction from your hypothesis?

Is your hypothesis technical? What does it forbid?

Testing my Conception of Science v1.0

I’ll test my idea of a hypothesis on the list of example “hypotheses” provided by Dr. Paul Strode.

It is hypothesized that a forefoot strike pattern will correlate with lower ground reaction forces.

What happens when you do some other strike pattern? Will you get lower or higher ground reaction? I can’t see a causal model here.

It is hypothesized that the structural and functional integrity of the system as a whole is dependent on nerve activity.

Cool. But that still doesn’t help me make any predictions. It is as vague as you can get. What happens if there is a lot of nerve activity? What happens if there is very little nerve activity?

We aimed to test the hypothesis of whether young healthy women will increase muscle mass and lose fat mass after undergoing 12 wk of intense resistance training (Josse et al. 2010).

This just a method followed by a prediction. Why would intense resistance training make you increase muscle mass and lose fat mass? What if I did light resistance training? Would I gain a lot more muscle or a lot less muscle? What if I did it for 50 weeks instead of 12? What about just 1 week? What results will I see? Your “hypothesis” does nothing to answer these questions.

If food is present in the aquarium, then snails will move with greater speed (toward the food) (Green 2004).

This is just a prediction. What if you place no food, just the smell of it? What if you place dummy food? Wait. Taboo “food”. What if you place a tasty burger? What if you place a rotting dog corpse? Will the snails still move with the same gusto?

Aspirin inhibits key oncogenic factors and/ or activates pivotal tumor suppressor genes.

Ah. This is getting closer and closer to a causal model. You can test this in several ways - make somebody eat aspirin, maybe inject them with aspirin (is that possible?), or do something else that reliably increases aspirin content in the brain. All of those should work.

If a plant receives fertilizer, then they will grow to be bigger than a plant that doesn t receive fertilizer.

This is a prediction, without a precise method or hypothesis. How much fertilizer? Of what kind? For how many days? What will happen if I change any of those parameters - will the plant grow more or less? You need to answer the “why” of it.

Marsh grass growth is limited by available nitrogen (Miller and Levine 2010).

This is close to a causal model. It predicts a lot - you can test it by keeping nitrogen level constant and increasing everything else (growth shouldn’t be affected, because nitrogen is the limiting factor); you can test it by increasing the nitrogen level. Of course, it would be better if it explained exactly how nitrogen affects growth, which might help us figure out the exact relation between growth and nitrogen level.

If the farmer burns the prairie then the next year will produce taller plants in his field then the previous year.

Just a method followed by prediction. Why is the burning of the prairie connected to the height of the plants? What if you burn some other part of the farm? What if you burn stuff, grow something else for a year, and then grow another crop - will they be taller than usual? And so on.

Based on this observation, we hypothesized that natural selection may have influenced AMY1 copy number in certain human populations (Perry et al. 2007).

Why is AMY1 present? Why isn’t it absent? Which specific human populations does your model predict will have this thing? Why them? Why not others? What sort of fitness advantage does it give so that it keeps propagating itself? Why is it not in everybody?

The fire is replenishing the nutrients in the soil.

This is a decent causal model, but not very precise. What kind of nutrients? What kind of fire? How does it depend on the duration of the fire? What if you pour water immediately afterwards? Does it take time for the nutrients to get “replenished”? Do you have to wait till you start planting the crops? Where exactly do the nutrients go? What if I removed the top layer of the soil (or have a flood that does the same)? Can you replenish the nutrients some other way?

The foraging patterns of S. carpocapsae, as measured by directional response, are affected by electrical fields.

Ok. But how? What if you keep changing the intensity of the electrical fields? Do they revert when you stop the electrical field? What about other kinds of radiation? What about other species? Does your causal model predict why their foraging pattern (of all things) might be affected by electrical fields?

I believe resin content of various pine species will affect its energy output.

Ok. But how exactly does the energy output vary with the resin content? Can you predict the relation across a wide variety of existing pine species? What if I introduce more resin into the tree? What if I remove the resin content (I’m talking out of my ass here, pardon me)? What if people genetically modify some pine species so that it has more resin content? Talking about cause, why do you predict that resin content will affect the energy output at all? What is your explanation for the link between the two?

What is my causal model of science?

I believe that thinking scientifically will help me a lot. But how? What exactly will I do and how exactly will it help me?

Ideas

Major attitude: everything has an explanation! Explain literally everything! Or, rather, anticipation-constrain literally everything! Figure out the cause behind everything. And, of course, when somebody claims that X is the cause, don’t accept it without evidence.

Don’t accept anything without evidence Somebody: “X is true” You: “So you say. Show me the evidence.”

A plan is based on a hypothesis (or, at least, on certain predictions) Successful plans are those that are based on correct hypotheses A plan is only as good as the hypothesis it is based on! Wait. Does this mean that a hypothesis drives a plan? Like, you could derive a plan from a hypothesis? And that, given one hypothesis, you can only have one possible (deterministic) plan? This would mean that choosing between plans is basically about choosing between hypotheses. i.e., Decision-making boils down to Truth-finding. But it always has! Remember, your Prediction Score determines everything! Corollary: Don’t ask which plan you should choose (among several), but rather, which hypothesis is true out of several. Ask not which plan would be most successful (out of several plans that achieve the same ends) Instead, ask what are the differing predictions between the hypotheses that those plans represent and see what the reality has been so far. Corollary: Don’t ask about the plan. Ask about the hypothesis it relies on. What are the assumptions it is based on? What does it expect will happen? Corollary: This implies you don’t just talk about a plan. You talk about multiple plans and choose between them. Remember, it’s meaningless to talk about just one hypothesis. Plans are programs. Hypotheses are programs.

Historical Perspective: Curiosity about how things came to be the way they are Why do we have supermarkets instead of ration shops? Why voting / democracy? Why civil service? Why parliament? These aren’t arbitrary facts. They are tightly governed by the true hypothesis about reality. They are superbly entangled. You shouldn’t be accepting it either way without question. “X happened? Oh, ok”. “X didn’t happen? Oh, ok.” – Wrong! Why does USA have Hollywood and why is it so popular around the world? It doesn’t have more people than Europe (I think)

Remember: Going into a new field (like Category Theory) is like going into the magical world. New rules to figure out and stuff. But the most important thing is that you’ve still got the experimental method, and that’s what matters.

The Scientific Method: In short, and in theory

Random Experiment => Generate Hypotheses using the possible outcomes

Outcome => Update on Evidence - enumerate the hypotheses that might matter

Generate Hypotheses => Look at the Random Experiments and their possible outcomes, look at the existing evidence, come up with Alternative Hypothesis, and Test

Also, Generate Experiment => Modify the state so that it distinguishes between two hypotheses

Information to maintain List of major hypotheses (per domain) List of special hypotheses List of evidence (per domain) List of strange evidence

Types of values you can find Events, Outcomes, Hypotheses, Predictions, and Domains. Domain = Set of Events, I think

Feedback: Get it from your predictions for various Random Experiments

Ideas for Scientific Method Domain (from HPMOR)

If Y is an important subject, test out hypotheses about its various properties How much low-hanging fruit is left? How much power does this concept have? etc.

If you come across a new fact (aka some event + outcome), Anticipation-Constrain it against your theories Is it not possible? Raise huge alarms. Do your theories make any interesting inferences about it?

If you come across a new fact (event + outcome), update on that evidence

If you come across some strange or awesome thing, ask if you know its causal structure

If you see X doing Y, ask why did X do some action Y. (X could be you)

If you see X have a property Y, ask why. (X could be you)

If you see event Y happen to X, ask why. (X could be you)

If someone gives you a new hypothesis, ask for evidence. aka some program / causal structure to make predictions about certain events

If someone claims to have property X, test it out. (including yourself)

If you suspect Y exists, test it out.

You think you have learned technique Y. Test it out.

You think you have learned concept Y. Test it out.

If you want to make Y happen, come up with hypotheses.

Can Y happen? Check out the predictions of that hypothesis / Test it out.

In some new / confusing area, why doesn’t ~Y happen?

If you see some object Y, where does Y come from?

If you strongly believe Y is true, ask what would it take to convince you otherwise? Always be looking for ways to test your beliefs.

If you see X do some awesome thing Y, ask how does X do Y.

If you see some fact of the world Y, ask what caused Y to be like that. example: historical stuff like democracy, markets, etc.
